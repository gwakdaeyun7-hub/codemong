# 2강 — 코딩의 표현 방법 (자연어 / 의사코드 / 순서도)

- **목표 학습 결과**: 문제를 코드로 옮기기 전에 자연어 → 의사코드 → 순서도 단계로 절차를 표현하는 흐름을 이해하고, 셋의 차이와 용도를 구분해 말할 수 있다.
- **prerequisite**: 1강 "파이썬 개요 & 개발환경" 학습 완료
- **목표 분량**: 약 184초 (180초 ±20% 범위 내)
- **타깃 시청자**: 파이썬 1강을 본 입문자. 코드 작성 경험은 아직 없음.
- **orientation**: landscape (1920x1080), 30fps
- **scene 수**: 13개

    - 우측 하단에 작은 회상 카드 (1강 mock): "1강 — 파이썬 환경 갖추기 ✓" (opacity 0.6)
- **note**: 도입. 1강과 동일한 차분한 학원 인강 톤. 1강 회상 카드로 시리즈 연속성 표현. 4단 구조 도입(0:00–0:20) 권장에 부합.

    사람은 머릿속으로 채워 읽지만, 컴퓨터는 빠진 단계를 알아서 추론하지 않거든요.

- **note**: 학습 목표 1번 진입 + 오개념 3번 (컴퓨터가 빠진 단계 추론할 거란 기대) 의 토대. "왜 필요한가" 를 먼저 다루는 게 이 영상의 1순위.

## Scene 10 — 의사코드로 옮기기 (오개념 1 정면 처리)

- **duration_sec**: 22
- **narration**: |
    이제 의사코드로 옮겨 봅니다. 형식을 좀 정돈하는 거예요.
    한 가지 짚을게요. 의사코드는 우리끼리 약속한 표기일 뿐, 파이썬이 읽지는 않습니다. 정해진 형식이 하나로 고정돼 있지도 않아요.
    그러니까 그대로 실행해 보려고 하지 않아도 됩니다.
- **on_screen**:
    - 화면 좌측 절반: 앞 scene 의 자연어 박스 (opacity 0.5 로 톤 다운)
    - 화면 우측 절반: "의사코드" 라벨이 붙은 박스 (코드 에디터 느낌이지만 어두운 배경 X — 흰 배경 + 모노스페이스 폰트)

    - 3) 만약 가격이 5천 원보다 크면:

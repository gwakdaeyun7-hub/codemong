# Lesson 2 — 코딩의 표현 방법 (자연어 / 의사코드 / 순서도)

> Python 기초 12강 중 2강. 영상 길이 180초(±20%), 한국어, 학원 인강 스타일(차분·구조화).
> 다음 강의: 3강 "변수와 자료형".

---

## 1. 행동 가능 학습 목표

이 영상이 끝났을 때 학습자는 다음을 할 수 있어야 합니다.

1. **"문제를 코드로 바꾸기 전에 표현 단계가 필요하다"는 흐름을 한두 문장으로 설명할 수 있다.**
   (자연어로 절차를 풀어 쓰고 → 의사코드로 형식을 정돈하고 → 필요하면 순서도로 분기·반복을 시각화한 뒤, 마지막에 코드로 옮긴다는 큰 그림.)
2. **자연어 / 의사코드 / 순서도가 각각 무엇이고 언제 쓰는지 구분해 말할 수 있다.**
   (자연어: 문제를 사람 말로 풀어쓰기. 의사코드: 코드 형식에 가깝게 정돈한 절차. 순서도: 절차의 흐름을 도형으로 시각화.)
3. **순서도의 기본 4기호를 보고 어떤 단계인지 구분할 수 있다.**
   (시작/끝 = 둥근 사각형, 처리 = 사각형, 판단 = 마름모, 입출력 = 평행사변형. 외우게 하기보다 "보면 알아본다" 수준.)
4. **주어진 자연어 절차 한 단락을 의사코드 형식으로 옮길 수 있다.**
   ("커피값을 계산해서 5천 원이 넘으면 카드, 아니면 현금으로 낸다" 같은 일상 절차를 줄 단위 의사코드로.)
5. **간단한 분기(조건이 하나 있는) 절차를 순서도로 그려본 결과를 읽고 잘못된 부분을 지적할 수 있다.**
   (직접 그리는 것보다, 잘못 그린 순서도를 보고 "판단 기호에서 화살표가 한 방향만 나가 있다" 같은 오류를 짚는 수준.)

> 이 강의는 **코드를 한 줄도 쓰지 않습니다.** 자연어·의사코드·순서도만으로 사고를 정리하는 단계.
> 실제 파이썬 문법 학습은 3강(변수와 자료형)부터.

## 2. Prerequisite

- **1강 "파이썬 개요 & 개발환경" 학습 완료 가정.**
  Python이 언어이고 에디터가 도구라는 구분, 코드를 쓰면 파일에 저장하고 실행한다는 흐름이 머릿속에 잡혀 있어야 합니다.
- 그 외 별도 사전 지식 없음. 코드 작성 경험은 필요하지 않습니다.
- 일상에서 "단계를 따라 무언가를 한다"는 감각(요리 레시피, ATM 사용 순서 등)만 있으면 충분.

## 3. 입문자가 자주 막히는 지점 / 오개념

2강 내용에서 새로 도출한 항목. 영상에서 짧게라도 짚고 넘어갈 가치가 있음.

1. **"의사코드는 진짜 코드인 줄 알고 그대로 실행해 보려 함"**
   - 의사코드는 *사람이 읽기 위한* 절차 정리이지, 컴퓨터가 실행하는 코드가 아님. 문법 검사도 없고 정답 형식도 하나로 고정돼 있지 않음.
   - 영상에서: 의사코드를 처음 보여줄 때 "이건 우리끼리 약속한 표기일 뿐, 파이썬이 읽지 않는다"고 한 문장으로 짚을 것.

2. **"순서도 화살표가 판단(마름모)에서 한 방향만 나간 것"**
   - 판단 기호는 항상 둘 이상의 분기(예/아니오)가 나가야 함. 한 방향만 그리면 흐름이 막힌 것과 같음.
   - 영상에서: 잘못된 예시 한 컷 + 올바른 예시 한 컷을 짧게 비교.

3. **"자연어로 적은 절차에 빠진 단계를 컴퓨터가 알아서 채워줄 거라 기대"**
   - 사람은 "당연히" 채워 읽지만, 컴퓨터는 빠진 단계를 추론하지 않음. "사인 안 하고 끝내면 결제가 안 된다"처럼 *암묵 단계도 명시해야* 코드로 옮길 수 있다는 감각.
   - 영상에서: 자연어 한 단락을 의사코드로 옮기는 예시에서 "이 단계 빠졌죠?"를 한 번 짚고 추가하는 흐름으로.

4. **"세 가지(자연어/의사코드/순서도) 중 무엇이 가장 좋은 표현인지" 물어봄**
   - 우열이 아니라 *용도가 다름*. 처음 문제를 풀어쓸 땐 자연어, 절차를 정돈할 땐 의사코드, 분기·반복이 복잡해 한눈에 보고 싶을 땐 순서도. 셋 다 안 쓰는 경우도 흔함.
   - 영상에서: 정리 구간에 한 문장으로 "셋 중 하나만 정답인 게 아니다"를 짚을 것.

> 4개 모두 짧게 다루되, 비중은 1번·2번에 무게를 두고 3번·4번은 한두 문장으로.

## 4. Scope 제한 (이 영상에서 다루지 *않을* 것)

영상 대본 작성 시 아래 주제는 의도적으로 제외합니다. 다른 강의에서 다룹니다.

- **실제 파이썬 코드 작성** — 3강 이후. 2강에선 코드 한 줄도 보여주지 않음(예고 차원의 짧은 한 컷도 자제).
- **변수, 자료형, `print()`, `input()`** — 3~4강.
- **조건문 / 반복문 문법** — 5~6강. 단, 순서도의 *판단 기호* 설명에서 "조건이 갈라진다"는 *개념*은 살짝 등장. 문법은 절대 안 들어감.
- **알고리즘 복잡도, Big-O, 시간/공간 효율** — 입문 코스 범위 밖.
- **순서도 기호 전체 목록(저장, 연결자, 처리부호 등)** — 4기호(시작/끝·처리·판단·입출력)만. 더 넣으면 인지 부하만 증가.
- **UML, 액티비티 다이어그램, 시퀀스 다이어그램 등 다른 표기법** — 입문 단계엔 불필요.
- **의사코드의 표준화된 문법(IF...THEN, FOR i FROM 1 TO N 등 영어 키워드 정형)** — 이 영상은 *한국어 의사코드*로만 보여줌. "정해진 형식이 하나는 아니다"는 점만 한 문장으로.
- **반복(loop) 구조의 순서도 표현** — 분기 하나만. 반복 표현은 6강에서 다룸.

## 5. 학습 흐름 권장 (대본 작성자 참고용)

영상의 4단 구조(도입 → 개념 → 예시 → 정리)에 맞춘 분배 권장. 시간은 권장이며, video-script-writer 가 narration 길이에 맞춰 조정 가능.

| 구간 | 시간(대략) | 다룰 내용 |
|------|-----------|----------|
| 도입 | 0:00 ~ 0:20 | "코드를 쓰기 전에 한 단계가 더 있다"는 문제 제기. 1강 복습 한 줄("환경은 갖췄다") + 오늘 다룰 세 가지 예고. |
| 개념 | 0:20 ~ 1:10 | 왜 표현 단계가 필요한지(목표 1) → 자연어 / 의사코드 / 순서도 각각의 정의와 용도(목표 2). 순서도 4기호도 이 구간에서 도입(목표 3). |
| 예시 | 1:10 ~ 2:30 | 일상 시나리오 한 개를 잡아 자연어 → 의사코드 → 순서도 순서로 변환(목표 4). 변환 도중 오개념 1·3 자연스럽게 삽입. 순서도 그릴 때 잘못된 분기 예시 한 컷으로 오개념 2 짚기(목표 5). |
| 정리 | 2:30 ~ 3:00 | 오늘 한 일 한 줄 요약 + 오개념 4(셋 중 하나만 정답이 아님) 한 문장 + 다음 강의 예고("이제 코드로 옮기기 위해, 변수와 자료형부터"). |

### 일상 시나리오 후보 (대본 작성자 선택용)

자연어 → 의사코드 → 순서도 변환 예시로 쓸 만한 한국 입문자 친화 시나리오:

- **편의점에서 음료 사기**: 가격 확인 → 5천 원 넘으면 카드, 아니면 현금. (가장 단순한 분기 1개.)
- **지하철 환승 판단**: 같은 호선이면 그대로, 다른 호선이면 환승 게이트로. (분기 1개.)
- **학번으로 학년 판단**: 입학 연도 빼서 학년 산출. (입출력·처리 기호가 자연스레 등장.)

위 셋 중 하나를 골라 한 영상 안에서 **한 시나리오만** 끝까지 변환. 시나리오를 두 개 이상 쓰면 시간도 부족하고 인지 부하만 증가.

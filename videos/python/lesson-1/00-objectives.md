# Lesson 1 — 파이썬 개요 & 개발환경

> Python 기초 12강 중 1강. 영상 길이 180초, 한국어, 학원 인강 스타일(차분·구조화).
> 다음 강의: 2강 "코딩의 표현 방법 — 자연어 / 의사코드 / 순서도".

---

## 1. 행동 가능 학습 목표

이 영상이 끝났을 때 학습자는 다음을 할 수 있어야 합니다.

1. **Python이 무엇이고, 왜 입문자에게 자주 쓰이는 언어인지 한두 문장으로 설명할 수 있다.**
   (사람이 읽기 쉬운 문법, 데이터·자동화·AI 등 폭넓은 활용 — 장점을 과장하지 않고 사실 수준에서.)
2. **자기 컴퓨터(Windows / macOS) 운영체제에 맞는 Python 3 설치 파일을 공식 사이트(python.org)에서 받아 설치할 수 있다.**
   (Windows 설치 시 "Add Python to PATH" 체크 포함.)
3. **터미널 / 명령 프롬프트에서 `python --version`(또는 `python3 --version`)을 입력해 설치된 Python 버전을 확인할 수 있다.**
4. **에디터(VS Code 권장)에 `.py` 파일을 만들어 저장하고, 저장한 파일을 터미널에서 실행하는 흐름을 따라 할 수 있다.**
   (코드 작성 자체가 아니라 "쓰기 → 저장 → 실행"의 순서 인식.)
5. **"Python(언어)"과 "에디터(VS Code 등 도구)"가 서로 다른 것임을 구분해 말할 수 있다.**

> 코드 작성·문법 학습은 3강(변수와 자료형)부터. 1강은 **"환경을 갖추고, 실행 흐름을 머리에 그릴 수 있게 하는"** 단계.

## 3. 입문자가 자주 막히는 지점 / 오개념

영상에서 짧게라도 짚고 넘어갈 가치가 있는 항목 — 1강 단계에서 실제로 자주 발생함.

1. **"Python = VS Code"라고 생각하는 혼동**
   - Python은 *언어*, VS Code는 *코드를 쓰는 도구(에디터)*. 둘은 별도 설치 대상이며 역할이 다름.
   - 영상에서: Python 설치 후 "이제 끝난 게 아니다, 코드를 쓰는 도구가 따로 필요하다"는 흐름으로 두 개념을 분리해서 보여줄 것.
2. **Windows 설치 시 "Add Python to PATH"를 체크하지 않아 터미널에서 `python` 명령이 안 먹는 문제**
   - 가장 흔한 1강 좌초 지점. 설치 화면을 보여줄 때 이 체크박스를 명확히 강조.
3. **에디터에서 코드를 수정한 뒤 저장(Ctrl/Cmd+S)을 안 하고 실행 → 변경사항이 반영 안 됨**
   - "파일은 저장돼야 실행에 반영된다"는 흐름을 명시. 5강 학습 목표 4번과도 연결.
4. **Python 2 vs Python 3 혼동 — 옛 자료의 `python2` 명령을 따라 했다가 동작이 다름**
   - 짧게 한 문장으로 정리: "지금은 Python 3을 씁니다. 2는 더 이상 권장되지 않습니다." (깊이 들어가지 말 것.)

> 4개 모두 다루되, 비중은 1번·2번에 무게를 두고 3번·4번은 한두 문장으로 짧게.

## 4. Scope 제한 (이 영상에서 다루지 *않을* 것)

영상 대본 작성 시 아래 주제는 의도적으로 제외합니다. 다른 강의에서 다룹니다.

- **변수, 자료형, `print()`** — 3강.
- **`input()` 및 연산자** — 4강.
- **조건문 / 반복문** — 5~6강.
- **실제 코드 작성·문법 설명 일체** — 1강은 "환경 준비"가 목적. 영상 끝에 한 줄짜리 데모(`print("Hello, Python")`)를 *실행 흐름 시연용*으로만 쓸 수 있고, 그 코드가 *왜* 그렇게 생겼는지는 설명하지 않음.
- **가상환경(venv) / 패키지 관리(pip install)** — 입문 1강 범위 밖. 이후 11강(파일 입출력)·외부 라이브러리 도입 시점까지 미룸.
- **IDE 심화 사용법, 단축키, 확장 프로그램 추천 목록** — 1강에선 "에디터를 연다 → `.py` 파일을 만든다 → 저장한다 → 실행한다"의 최소 동선만.
- **다른 에디터/IDE 비교 (PyCharm, Jupyter, Cursor 등)** — VS Code 하나로 통일해 보여줌. 선택지를 늘리면 입문자 인지 부하만 늘어남.
- **운영체제별 차이의 모든 케이스** — Windows를 메인 흐름으로, macOS는 한 줄 보충 정도. Linux는 1강에선 다루지 않음.

## 5. 학습 흐름 권장 (대본 작성자 참고용)

영상의 4단 구조(도입 → 개념 → 예시 → 정리)에 맞춘 분배 권장:

| 구간 | 시간(대략) | 다룰 내용 |
|------|-----------|----------|
| 도입 | 0:00 ~ 0:20 | 이 영상에서 무엇을 하게 되는지 한 줄 요약. 과장된 hook 금지. |
| 개념 | 0:20 ~ 1:00 | Python이 무엇인지(목표 1), 언어와 에디터의 차이(목표 5). |
| 예시 | 1:00 ~ 2:30 | 설치(목표 2) → 버전 확인(목표 3) → `.py` 파일 작성·저장·실행 흐름(목표 4). 오개념 1·2·3을 이 구간에 자연스럽게 삽입. |
| 정리 | 2:30 ~ 3:00 | 오늘 한 일 한 줄 요약 + 오개념 4 한 줄 + 다음 강의 예고("다음은 코드를 쓰기 전, 코딩의 표현 방법부터"). |

> 위 시간 분배는 권장이며, 대본 작성자(video-script-writer)가 실제 narration 길이에 맞춰 조정 가능.

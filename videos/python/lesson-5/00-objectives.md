# Lesson 5 — 조건문 (`if` / `elif` / `else` / 중첩 조건문)

> Python 기초 12강 중 5강. 영상 길이 180초(±20%), 한국어, 학원 인강 스타일(차분·구조화).
> 이전 강의: 4강 "입력과 연산자". 다음 강의: 6강 "반복문".
>
> **중요 컨텍스트** — 4강 영상은 아직 미제작 상태이지만, **5강 영상은 4강 학습 완료를 전제**합니다. 단, 비교 연산자(`==`, `>`, `<` 등)는 5강에서 *짧게 복습*하고 들어갑니다(조건식 없이 if를 다룰 수 없으므로). 산술/논리 연산자의 깊은 다룸은 4강 몫이라 5강에서는 일부러 깊게 들어가지 않습니다.

---

## 1. 영상 한 줄 요약

이 영상이 끝났을 때 학습자는 **주어진 상황에서 "조건에 따라 어느 코드를 실행할지"를 `if` / `elif` / `else` 로 갈래 지어 작성한 코드를 읽고, 어느 분기가 실행될지 짚을 수 있다.**

---

## 2. 행동 가능 학습 목표

이 영상이 끝났을 때 학습자는 다음을 할 수 있어야 합니다.

> 5강은 학습자가 처음으로 "프로그램의 흐름이 갈라지는" 코드를 만나는 강의입니다. 도착점은 "if 한 줄을 외운다"가 아니라 "이 if 블록을 읽고 어느 줄이 실행될지 손가락으로 가리킬 수 있다".

1. **조건문이 왜 필요한지 한 문장으로 설명할 수 있다.**
   ("어떤 값이냐에 따라 코드를 다르게 실행하기 위해서." 2강의 순서도에서 마름모로 그려졌던 분기가 코드에서는 `if` 가 된다는 연결을 인지.)
2. **`if 조건: / 들여쓰기된 코드` 한 블록을 보고, 조건이 `True`일 때만 들여쓰기된 코드가 실행된다는 것을 설명할 수 있다.**
   (예: `if score >= 60:` 아래 들여쓰기된 `print("합격")` 한 줄을 가리키며 "이 줄은 조건이 참일 때만 실행"이라고 말할 수 있음.)
3. **`if` / `elif` / `else` 가 들어간 3분기 코드를 읽고, 주어진 입력값에 대해 어느 분기가 실행되는지 골라낼 수 있다.**
   (성적 등급(A/B/C/F) 예제에서 `score = 75` 일 때 어느 분기에 도달하는지 손으로 따라갈 수 있음. **위에서부터 차례로 검사하다가 처음 참이 되는 분기에서 멈춘다**는 흐름을 인지.)
4. **`==` 와 `=` 의 차이를 한 문장으로 구분해 말할 수 있다.**
   ("`=`는 값을 변수에 _넣을 때_, `==`는 두 값이 _같은지 비교할 때_." 5강의 가장 핵심적인 표기 구분.)
5. **2단 중첩 조건문을 보고, 바깥 조건과 안쪽 조건이 _모두_ 참일 때만 안쪽 코드가 실행됨을 설명할 수 있다.**
   (예: 바깥 `if 로그인 됨:` 안에 다시 `if 관리자:` 가 들어간 구조. 들여쓰기 깊이가 곧 "이 블록은 어느 조건에 속하는지"를 의미함을 인지.)

> 영상에서 학습자가 코드를 직접 타이핑하지는 않습니다. **읽고 어느 분기로 흐르는지 따라갈 수 있음** 수준이 도착점.

---

## 3. Prerequisite

- **3강 "변수와 자료형" 학습 완료 가정.**
  변수에 값이 담겨 있다는 감각, 자료형 중 **불린(`True` / `False`)** 의 첫 글자 대문자 표기를 알고 있어야 합니다. 조건식의 결과는 결국 불린이라는 연결고리가 5강 전반의 토대가 됩니다.
- **4강 "입력과 연산자" 학습 완료 가정 (단, 영상 미제작 — 5강에서 짧게 복습).**
  비교 연산자(`==`, `!=`, `>`, `<`, `>=`, `<=`)가 두 값을 비교해 `True` / `False`를 만들어낸다는 점을 알고 있어야 합니다. 5강에서는 첫 1컷에서 이를 한 줄로 복습한 뒤 진행합니다. 논리 연산자(`and` / `or` / `not`)는 5강 본문에선 다루지 않고, 두 조건을 동시에 검사할 때는 **중첩 if** 로만 처리합니다(4강이 완성되면 다음 차수 영상부터 `and`/`or` 도 자연스레 사용 가능).
- **2강 "코딩의 표현 방법" 학습 완료 가정.**
  순서도에서 마름모(분기)가 어떤 모양·역할이었는지 기억하고 있으면 도입에서 if와 매끄럽게 연결할 수 있습니다.
- 그 외 별도 사전 지식 없음. 다른 언어 경험은 필요하지 않습니다.

---

## 4. 입문자가 자주 막히는 지점 / 오개념

5강은 **표기 디테일에서 막히는 비중이 가장 큰 강의**입니다. 비중을 1번·2번에 무겁게 두고, 나머지는 한 컷씩 짧게.

1. **`=` 와 `==` 혼동 — `if score = 60:` 으로 쓰려고 함.**
   - 수학에서 등호(`=`)는 "같다"의 의미였기 때문에 비교에도 `=` 한 개를 쓰려는 경향이 매우 강함. Python에서 `=`는 _대입_, `==`는 _비교_ 로 완전히 다른 일을 함.
   - 영상에서: 조건식을 처음 보여주는 컷에서 `==` 두 개를 큰 글자로 한 번 강조하고, "`=`는 값 넣을 때, `==`는 같은지 물어볼 때" 한 줄로 정리. 깊게 안 들어감.

2. **`if` 다음 줄 들여쓰기 누락 또는 잘못된 깊이.**
   - `if score >= 60:` 다음 줄을 들여쓰지 않거나, 한 칸 / 두 칸 / 네 칸이 섞임. Python에서 들여쓰기는 *문법*이기 때문에 들여쓰기가 틀리면 코드 자체가 동작하지 않거나 의도와 다르게 실행됨.
   - 영상에서: if 블록의 첫 등장 컷에서 들여쓰기된 줄을 시각적으로 살짝 들여 보여주고 "이 들여쓰기 한 단계가 'if에 속한 줄'이라는 표시"라고 한 문장 명시. 들여쓰기 칸 수(스페이스 4칸 권장)는 별도로 강조하지 않음 — 이 영상에서는 *깊이가 같다*는 시각적 일치만 보이면 충분.

3. **`elif` 가 아니라 `else if` 라고 쓰려고 함.**
   - 다른 언어(JavaScript, Java, C 등) 경험이 있거나 영어 어순에 익숙한 학습자는 자연스럽게 `else if` 라고 쓰려 함. Python은 **`elif` 한 단어**로만 인정됨.
   - 영상에서: 3분기 예제를 처음 띄울 때 `elif` 키워드 위에 짧은 콜아웃 "Python에선 `elif` 한 단어로 씁니다" 한 줄. 한 컷 안에서 처리.

4. **`if` 두 개를 나란히 쓰면 `if-elif`와 똑같이 동작할 거라고 기대.**
   - `if 조건A: ... if 조건B: ...` 와 `if 조건A: ... elif 조건B: ...` 는 의미가 다름. 앞은 두 if가 _각각 독립적으로_ 검사돼 둘 다 참이면 둘 다 실행. 뒤는 위에서부터 차례로 검사해 *처음 참이 되는 한 분기*만 실행하고 나머지는 건너뜀.
   - 영상에서: 시간이 빠듯하므로 깊이 다루지는 않되, **3분기 예제(A/B/C 등급)** 흐름을 짚을 때 "위에서부터 차례로 검사, 한 곳에 걸리면 끝" 한 문장을 명시해서 이 오개념을 자연스럽게 차단. 직접 `if-if` 반례를 보여주진 않음(인지 부하 증가).

5. **중첩 if의 들여쓰기 깊이를 손으로 못 따라감 — "이 줄은 어느 if에 속한 거지?"**
   - 들여쓰기가 두 단계 들어간 줄이 안쪽 if에 속한 줄인지, 바깥 if에 속한 줄인지 헷갈림. 들여쓰기 *깊이*가 곧 *어느 블록 소속이냐*임을 시각적으로 잡아주는 컷이 필수.
   - 영상에서: 중첩 예제 한 컷에서 들여쓰기 단계마다 다른 배경색 띠나 세로 가이드 라인(remotion-composer 가 처리)을 그어 "이 띠 안의 줄들이 같은 블록"임을 시각적으로 보여줄 것.

> 1·2·5번에 무게를 두고, 3·4번은 한 컷·한 문장으로 짧게.

## 5. Scope 제한 (이 영상에서 다루지 _않을_ 것)

영상 대본 작성 시 아래 주제는 의도적으로 제외합니다. 다른 강의에서 다루거나 입문 단계 밖입니다.

- **논리 연산자 `and` / `or` / `not` 의 본격 사용** — 4강 몫. 5강에선 두 조건을 동시에 검사할 때 **중첩 if** 로만 처리합니다. (4강이 만들어지면 5강의 후속 응용 영상에서 자연스레 등장.)
- **`match` / `case` 문 (Python 3.10+)** — 다중 분기의 대안이지만 입문 단계에서는 `if/elif/else` 만으로 충분. 별도 강의도 잡지 않음(입문 코스 범위 밖).
- **삼항 표현식 / 조건 표현식 (`A if 조건 else B`)** — 한 줄 분기. `if/elif/else` 의 기본형이 익숙해진 다음 단계. 5강에서는 다루지 않음.
- **단축 평가(short-circuit evaluation) 동작 디테일** — `and` / `or` 가 좌측에서 결정되면 우측을 평가하지 않는다는 내부 동작. 4강 이후 응용 단계.
- **불린 변환 규칙 / "falsy" 값 목록(`0`, `""`, `[]`, `None` 이 `if` 조건에선 거짓 취급)** — 입문자에게 인지 부하 큼. 5강에선 조건식은 항상 _명시적인 비교_(`score >= 60` 등) 만으로 작성. 암묵 변환은 다루지 않음.
- **`elif` 체인의 가독성 가이드(분기가 많을 때 함수 분리 / 딕셔너리 매핑 등)** — 코드 설계 영역. 입문 코스 범위 밖.
- **3단 이상의 깊은 중첩** — 5강은 **2단 중첩까지만**. 3단 이상은 가독성 문제이자 별도 리팩토링 주제.
- **예외 처리(`try` / `except`)** — 별도 강의. 잘못된 입력에 대한 "방어"는 5강 본문에서 다루지 않음.
- **`is` vs `==` 의 차이** — 객체 동일성 비교. 입문 단계에서는 비교는 `==` 만으로 충분. 별도 강의 몫.
- **부동소수점 비교 함정(`0.1 + 0.2 == 0.3` 이 `False` 인 이유)** — 5강에서는 정수 비교만 사용. 부동소수점 비교 자체를 회피해 함정을 피합니다.

## 6. 3분 안에서의 시간 배분 권고

영상의 4단 구조(도입 → 개념 → 중첩 → 정리)에 맞춘 분배 권장. 시간은 권장이며, video-script-writer 가 narration 길이에 맞춰 조정 가능.

| 구간                   | 시간(대략)  | 분량 비중 | 다룰 내용                                                                                                                                                                                                                                                                 |
| ---------------------- | ----------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 도입                   | 0:00 ~ 0:20 | ~11%      | 2강의 순서도 마름모를 잠깐 떠올리고, "프로그램이 *판단*해서 길이 갈라지는 지점"을 코드로 옮긴 것이 조건문임을 한 줄 전환. 오늘 다룰 흐름(`if` → `elif`/`else` → 중첩) 예고. 비교 연산자(`==`, `>=`)는 _결과가 `True`/`False`_ 라는 점만 한 줄로 짧게 복습.                |
| 개념 — `if` 단독       | 0:20 ~ 0:55 | ~19%      | 가장 단순한 형태 1분기. `if 조건: / (들여쓰기) 실행할 줄` 의 시각 구조. 목표 1·2 달성. 오개념 1(`=` vs `==`) · 오개념 2(들여쓰기) 를 이 구간에서 처리.                                                                                                                    |
| 개념 — `elif` / `else` | 0:55 ~ 1:45 | ~28%      | 3분기로 확장. 성적 등급(A/B/C 또는 합격/재시험/불합격) 예제. 위에서부터 차례로 검사 → 처음 참인 분기에서 멈춤 → 어느 분기에도 안 걸리면 `else`. 목표 3 달성. 오개념 3(`elif` 한 단어) · 오개념 4(`if-if` ≠ `if-elif`) 를 이 구간에서 처리. **5강에서 가장 비중 큰 구간.** |
| 중첩                   | 1:45 ~ 2:35 | ~28%      | 바깥 if 안에 다시 if. **2단까지만**. 예: 로그인 여부(바깥) → 관리자 여부(안쪽). 들여쓰기 깊이가 곧 "어느 블록 소속"임을 시각적으로 보여줌. 목표 5 달성. 오개념 5(중첩 깊이 추적) 가 이 구간 메인.                                                                         |
| 정리                   | 2:35 ~ 3:00 | ~14%      | "조건에 따라 코드가 갈라진다 / `if` → `elif` → `else` 는 위에서부터 차례로 / 중첩은 들여쓰기 깊이가 곧 소속" 세 줄 요약. 다음 강의 hook(6강 "반복문").                                                                                                                    |

### 일상 시나리오 후보 (대본 작성자 선택용)

조건문 분기를 한국 입문자에게 가장 자연스럽게 보여주는 시나리오:

- **시험 점수 → 합격/재시험/불합격** (또는 A/B/C/F 등급): 점수 한 개의 *연속 구간*을 세 분기로 나누는 가장 정석적인 예제. 비교 연산자 `>=` 가 자연스럽게 등장하고, "위에서부터 차례로 검사"의 흐름이 직관적으로 보임. **본문 3분기 예제로 추천.**
- **나이 → 어린이/청소년/성인 요금**: 점수와 동일한 구조. 학습자(초·중학생) 연령대에 자기 자신이 어느 분기에 들어가는지 즉시 확인 가능해 몰입도 좋음. 대안.
- **로그인 여부 → (참일 때만) 관리자 여부**: 이쪽은 **중첩 예제**로 적합. 바깥 조건이 참일 때만 안쪽 조건을 따져야 한다는 구조가 자연스럽게 드러남. **중첩 구간 예제로 추천.**

> 본문 3분기 = **시험 점수 → 합격/재시험/불합격**, 중첩 = **로그인 → 관리자**. 두 시나리오를 분리하면 한 시나리오에 하나의 조건문 형태만 등장해 인지 부하가 낮아집니다. video-script-writer 가 두 시나리오의 시각적·서사적 결을 보고 최종 선택.

> 한 시나리오에 너무 많은 분기를 욱여넣지 말 것. 3분기 + 2단 중첩 이상은 3분 안에서 처리 불가능.

## Scope 위반 여부

> 기준: 00-objectives §5. 다루지 않을 항목들이 정말 등장하지 않는가.

| 항목                               | 등장 | 비고                                                                                                                                |
| ---------------------------------- | ---- | ----------------------------------------------------------------------------------------------------------------------------------- |
| 논리 연산자 `and`/`or`/`not`       | 없음 | 두 조건 동시 검사는 scene-12 중첩 if 로만 처리                                                                                      |
| `match`/`case`                     | 없음 | —                                                                                                                                   |
| 삼항 표현식 (`A if C else B`)      | 없음 | —                                                                                                                                   |
| 단축 평가 동작 디테일              | 없음 | —                                                                                                                                   |
| Falsy 값 (`0`, `""`, `[]`, `None`) | 없음 | 조건식이 모두 명시적 비교 (`score >= 60`) 또는 불린 직접 (`is_login`/`is_admin` 변수가 True/False 불린 직접 보유 → falsy 함정 회피) |
| 3단 이상 깊은 중첩                 | 없음 | scene-12 가 정확히 2단까지                                                                                                          |
| 예외 처리 (`try`/`except`)         | 없음 | —                                                                                                                                   |
| `is` vs `==`                       | 없음 | —                                                                                                                                   |
| 부동소수점 비교                    | 없음 | 모든 숫자 정수 (60, 75, 45, 40)                                                                                                     |

**모든 항목 준수.** 특히 falsy 회피는 미세하게 잘 잡혀 있음 — `if is_login:` 같은 표현이 등장하지만, scene-13 변수 박스에서 `is_login = True` 로 *불린 변수임을 명시*해 입문자가 "조건이 False여도 if가 작동?" 같은 falsy 함정으로 들어가지 않게 처리.

   - 다만 scene-11 에서 같은 mental model 을 단독 5.95초 노출 → scene-12 의 우측 박스는 _재확인용_ 으로 충분.
   - 6강·7강에서 mental model 도식 사용 시 좌측 코드 등장 _직후_ (5~7초 영역) 에 도식이 같이 등장하도록 timing 조정 권장. 5강 자체는 변경 불필요.

3. **Scene-09 흐름 화살표 길이** (강도: 참고).
   - 현재 FlowArrow가 startY=92, endY=200 (108px, 1줄~2줄까지만 진행). narration "위에서부터 차례로 검사 → 첫 줄에서 True → 멈춤" 의도라 적절하나, 일부 학습자는 "왜 화살표가 짧게 끝났지? 3~6줄로 안 가나?" 의문 가질 수 있음.

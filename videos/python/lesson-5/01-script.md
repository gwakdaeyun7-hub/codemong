# 5강 — 조건문 (`if` / `elif` / `else` / 중첩 조건문)

- **목표 학습 결과**: `if` / `elif` / `else` 로 갈래 지은 코드를 보고 주어진 입력값에 어느 분기가 실행될지 짚을 수 있고, 2단 중첩 조건문에서 들여쓰기 깊이가 어느 블록 소속을 의미하는지 설명할 수 있다.
- **prerequisite**: 3강 "변수와 자료형" 학습 완료 (불린·변수 감각) + 2강 "코딩의 표현 방법" 학습 완료 (순서도 마름모). 4강 "입력과 연산자"는 영상 미제작 상태이며, 비교 연산자(`==`, `>=`)는 본 강의에서 한 줄로 짧게 복습.
- **목표 분량**: 약 188초 (180초 ±20% 범위 내)
- **타깃 시청자**: 파이썬 1·2·3강을 본 입문자. 변수와 불린이 무엇인지는 알고 있으나, "프로그램의 흐름이 갈라지는" 코드는 이번 강의에서 처음 만남.
- **orientation**: landscape (1920x1080), 30fps
- **scene 수**: 14개
- **시나리오**: 본문 3분기 = 시험 점수 → 합격/재시험/불합격, 중첩 = 로그인 → 관리자

## Scene 5 — 오개념 직격: `=` vs `==`

- **sceneId**: scene-05
- **duration_sec**: 15
- **narration**: |
  여기서 가장 헷갈리는 표기 하나만 짚을게요.
  수학 시간에 등호는 "같다"였지만, 파이썬에서는 다릅니다.
  등호 한 개는 값을 변수에 _넣을 때_, 등호 두 개는 두 값이 _같은지 물어볼 때_ 씁니다.
- **on_screen**:
  - 화면 좌우 분할 비교 컷
  - 화면 좌측: 큰 글자로 `=`
  - 그 아래 코드 `score = 60` (라벨 "값을 넣는다", 회색)
  - 좌측 박스 외곽선 회색
  - 화면 우측: 큰 글자로 `==` (violet-500, 살짝 크게)
  - 그 아래 코드 `if score == 60:` (라벨 "같은지 비교한다", violet-500)
  - 우측 박스 외곽선 violet-500
  - 두 박스 사이 회색 구분선
  - 화면 하단 lower-third: "`=` 는 대입 · `==` 는 비교"
- **note**: 오개념 1번 (00-objectives §4) 정면 처리 — 비중 있게. 학습 목표 4번 달성. 좌우 분할 컷은 1강 scene-05, 3강 scene-06 와 동일 패턴 — 시즌 통일성. "등호" 라는 한국어 용어로 풀어 설명. 깊게 들어가지 않고 한 문장으로 정리 (00-objectives §4 권고 — 한 컷 안에서 처리).

### Scope 제한 준수 확인 (00-objectives §5)

- [x] `and` / `or` / `not` — 등장하지 않음. 두 조건 동시 검사는 중첩 if 로만 처리 (scene 12).
- [x] `match` / `case` — 등장하지 않음.
- [x] 삼항 표현식 — 등장하지 않음.
- [x] 단축 평가 — 등장하지 않음.
- [x] falsy 값 — 등장하지 않음. 조건식은 항상 명시적 비교 (`score >= 60` 등).
- [x] 3단 이상 중첩 — 등장하지 않음. 본 대본은 2단까지만 (scene 12).
- [x] 예외 처리 — 등장하지 않음.
- [x] `is` vs `==` — 등장하지 않음.
- [x] 부동소수점 비교 — 등장하지 않음. 모든 점수·조건은 정수만 사용 (60, 75, 45, 40).

### 발음 사전 적용 확인

본 강의는 조건문 키워드(`if`/`elif`/`else`)와 변수 이름(`score`/`is_login`/`is_admin`)이 다수 등장. narration 안 영어 토큰 검수:

- `Python` → "파이썬" ✓ (scene 1, 5, 8 narration — 사전 기존)
- `True` → "트루" ✓ (scene 3, 4, 6, 9, 10, 13 narration — 사전 기존)
- `False` → "폴스" ✓ (scene 3, 10, 13 narration — 사전 기존)
- `if` → "이프" ✓ (scene 4, 8, 9, 12, 13, 14 narration — **사전 신규 추가**)
- `elif` → "엘리프" ✓ (scene 8, 9, 14 narration — **사전 신규 추가**)
- `else` → "엘스" ✓ (scene 8, 9, 13, 14 narration — **사전 신규 추가**)
- `else if` → "엘스 이프" ✓ (scene 8 narration — **사전 신규 추가**, 오개념 3 반례 언급용)
- `score` → "스코어" ✓ (scene 6, 9, 10 narration — **사전 신규 추가**)
- `admin` → "어드민" ✓ (scene 12 narration — **사전 신규 추가**)

**한국어 토큰이지만 TTS 가독성 검수가 필요한 항목**:

- 코드 안 숫자 — narration 에서는 한자식 ("육십", "칠십오", "사십오", "사십") 으로 명시 (scene 3, 6, 7, 9, 10). 화면 텍스트는 아라비아 숫자 (`60`, `75`, `45`, `40`) 로 노출. 3강 scene-10·11 동일 정책.
- "관리자" / "일반 화면" — 한국어 단어, TTS 표준.
- "들여쓰기" / "갈래" / "분기" — 한국어 단어, TTS 표준.

**사전 미등록 처리한 단어**:

- `is_login`, `is_admin` — 변수 이름이라 매 영상마다 달라짐. 3강의 `is_student` 와 동일 정책 (사전 등재 X, narration 에 영문 그대로 두지 않고 한국어로 풀어 설명). 본 대본에서는 narration 에서 "로그인 여부" / "어드민 여부" 로 _풀어 설명_ — `is_login`/`is_admin` 자체를 발음으로 부르지 않으므로 사전 미등록.
- `print` — 본 대본 narration 에서 직접 호출되지 않음. 코드 화면에만 등장 (`print("합격")` 등). 3강 scene-10 에서 이미 "프린트 괄호" 로 도입 완료된 mental model 의 재활용이므로 narration 재호출 불필요.
- `:` (콜론) — narration 에서는 "콜론" 한국어로 풀어 씀 (scene 4). 사전 매핑 불필요.

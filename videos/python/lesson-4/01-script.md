## Scene s03 — 오개념 1: `input()` 반환은 항상 문자열

- **sceneId**: `s03`
- **durationSeconds**: 35

**narration**:
여기서 가장 자주 막히는 지점입니다. 인풋 함수가 돌려주는 값은 _항상_ 문자열입니다. 사용자가 숫자 3을 입력해도, 변수에 들어가는 건 따옴표가 붙은 문자 3입니다. 그래서 이런 일이 생깁니다. 첫째 값과 둘째 값을 받아서 더해 봅니다. 사용자가 3과 5를 입력했을 때, 우리가 기대한 건 8이지만 화면에는 35가 찍힙니다. 왜냐하면 문자 3과 문자 5를 더하는 건, 두 문자를 _이어붙이는_ 동작이기 때문입니다. 이걸 진짜 숫자로 다루려면, 인트 괄호로 한 번 감싸 줍니다. 인트는 정수를 뜻하는 함수로, 문자 3을 숫자 3으로 바꿉니다. 이렇게 바꾼 뒤 더하면, 화면에 8이 찍힙니다.

**visual**:

- 0~3s: 좌상단에 작은 라벨 카드 "자주 막히는 지점 1번 / 인풋 함수의 반환 타입" (violet-100 배경, violet-700 텍스트).
- 3~8s: 중앙에 코드 블록 type-on:
  ```
  a = input("첫째: ")
  b = input("둘째: ")
  print(a + b)
  ```

  - 각 줄당 1.5초 type-on.
- 8~13s: 우측 콘솔에 사용자 입력 시뮬레이션: `첫째: 3` → `둘째: 5` → 결과 `35` 가 큰 글씨 violet-500 으로 fade-in. 결과 옆에 빨간 ❌ 작은 마크와 라벨 "기대: 8 / 실제: 35" (단, 이모지는 사용 X — 단순 텍스트 라벨 "기대: 8, 실제: 35"). 라벨은 small caps 느낌의 작은 글씨.
- 13~18s: 화면이 좌우 둘로 split. 좌측 라벨 "문자 + 문자 = 이어붙이기", 좌측 코드 `"3" + "5"` 결과 `"35"`. 우측 라벨 "숫자 + 숫자 = 더하기", 우측 코드 `3 + 5` 결과 `8`. 둘은 좌→우 순서로 fade-in.
- 18~25s: split 화면이 다시 단일 화면으로 합쳐지며, 위 3줄 코드가 변형된다. `a = input("첫째: ")` 가 `a = int(input("첫째: "))` 로 두 부분이 추가 type-on (`int(` 와 `)` 가 violet-400 으로 부드럽게 끼어드는 애니메이션). 같은 처치를 `b` 줄에도 적용.
- 25~32s: 우측 콘솔이 갱신 — 같은 입력 3, 5에 대해 결과 `8` 이 violet-500 으로 크게 fade-in. 옆에 라벨 "기대: 8, 실제: 8".
- 32~35s: 화면 하단에 결론 카드 fade-in: "인풋의 결과는 문자열 — 숫자로 쓰려면 int() 로 감싼다".

**onScreenCode**:

```python
# 인풋의 결과는 항상 문자열
a = input("첫째: ")
b = input("둘째: ")
print(a + b)
# 입력: 3, 5
# 출력: 35  (문자열 이어붙이기)

# 숫자로 쓰려면 int() 로 감싼다
a = int(input("첫째: "))
b = int(input("둘째: "))
print(a + b)
# 입력: 3, 5
# 출력: 8
```

**emphasisBeats**:

- narration 의 "_항상_ 문자열입니다" 구절에서 좌상단 라벨 카드의 "문자열" 단어가 violet-500 으로 펄스 (한 번).
- narration 의 "8이 찍힙니다" 구절에서 콘솔의 `8` 결과가 0.3초 scale up 후 복귀.

## Scene s04 — 산술 연산자 7종 표 + 우선순위 한 문장

- **sceneId**: `s04`
- **durationSeconds**: 15

**narration**:
이제 계산에 쓰는 산술 연산자 일곱 가지를 한번에 보겠습니다. 더하기, 빼기, 곱하기, 나누기 네 개는 학교에서 본 그대로입니다. 슬래시 두 개는 몫만, 퍼센트는 나머지, 별표 두 개는 거듭제곱입니다. 그리고 한 가지 규칙. 곱셈과 나눗셈이 덧셈과 뺄셈보다 먼저 계산됩니다. 헷갈리면 괄호로 묶어 주세요.

**visual**:

- 0~6s: 화면 중앙에 7행짜리 표가 위에서 아래로 한 행씩 0.5초 간격으로 fade-in. 각 행은 [연산자 / 의미 / 예시 / 결과] 4열. 헤더는 zinc-700 배경 흰 글씨, 본문 행은 흰 배경.
  | + | 더하기 | 3 + 5 | 8 |
  | − | 빼기 | 7 − 2 | 5 |
  | _ | 곱하기 | 4 _ 3 | 12 |
  | / | 나누기 | 7 / 2 | 3.5 |
  | // | 몫 | 7 // 2 | 3 |
  | % | 나머지 | 7 % 2 | 1 |
  | ** | 거듭제곱 | 2 ** 3 | 8 |
  - 표 안의 연산자 글자(`+`, `-`, `*`, `/`, `//`, `%`, `**`) 는 모두 violet-500.
- 6~10s: 표 하단에 작은 분리선과 함께 한 줄 카드 fade-in: "곱셈·나눗셈이 덧셈·뺄셈보다 먼저. 헷갈리면 괄호."
- 10~15s: 화면 우측에 작은 코드 카드 두 개가 위아래로 등장. 위 카드 `2 + 3 * 4` → `14`, 아래 카드 `(2 + 3) * 4` → `20`. 두 결과 숫자(14, 20) 가 violet-500 으로 강조.

**onScreenCode**:

```python
# 산술 연산자 7종
3 + 5    # 8
7 - 2    # 5
4 * 3    # 12
7 / 2    # 3.5
7 // 2   # 3
7 % 2    # 1
2 ** 3   # 8

# 우선순위 한 줄
2 + 3 * 4    # 14
(2 + 3) * 4  # 20
```

```python
print(2 + 3 * 4)
# 출력: 14
```

**activeRecallBeat**:

## Scene s06 — 비교 연산자 6종 + 결과가 _값_

- **sceneId**: `s06`
- **durationSeconds**: 15

**narration**:
다음은 비교 연산자입니다. 두 값을 견주는 데 쓰는 여섯 가지. 같다, 다르다, 크다, 작다, 크거나 같다, 작거나 같다. 여기서 중요한 점 하나. 비교의 결과는 그 자체로 _값_ 입니다. 트루, 또는 폴스 두 개 중 하나입니다. 예를 들어, 3이 5보다 크냐고 묻고 프린트로 출력해 보면, 화면에는 폴스가 찍힙니다.

**visual**:

- 0~6s: 화면 좌측에 비교 연산자 6행짜리 표 fade-in (s04 의 표와 같은 디자인 톤):
  | == | 같다 | 3 == 3 | True |
  | != | 다르다 | 3 != 5 | True |
  | > | 크다 | 3 > 5 | False |
  | < | 작다 | 3 < 5 | True |
  | >= | 크거나 같다 | 3 >= 3 | True |
  | <= | 작거나 같다 | 5 <= 3 | False |
  - 연산자 기호는 violet-500. `True` / `False` 결과는 굵게.
- 6~10s: 우측에 코드 카드 type-on: `print(3 > 5)`. 그 아래 콘솔 박스에 결과 `False` 가 fade-in (큰 글씨, violet-600).
- 10~15s: `False` 글자 옆에 라벨 카드 fade-in: "비교의 결과는 True / False 라는 _값_". 라벨의 "값" 글자만 violet-500 으로 강조.

**onScreenCode**:

```python
# 비교 연산자 6종
3 == 3   # True
3 != 5   # True
3 > 5    # False
3 < 5    # True
3 >= 3   # True
5 <= 3   # False

# 비교 결과는 그 자체로 값
print(3 > 5)
# 출력: False
```

## Scene s07 — `=` vs `==` 한 컷

- **sceneId**: `s07`
- **durationSeconds**: 10

**narration**:
헷갈리기 쉬운 한 가지. 등호 하나는 *값을 담는다*는 뜻입니다. 등호 두 개는 *같은지 묻는다*는 뜻입니다. 모양은 비슷해도 역할은 전혀 다릅니다.

**visual**:

- 0~5s: 화면 좌우로 split. 좌측 라벨 "`=` 담는다", 그 아래 코드 `name = "지윤"` 과 작은 화살표 다이어그램 (변수 박스에 값이 들어가는 모션).
- 5~10s: 우측 라벨 "`==` 같은지 묻는다", 그 아래 코드 `name == "지윤"` 과 결과 박스 `True`. 두 영역의 등호 기호가 violet-500 으로 굵게 강조.

**onScreenCode**:

```python
name = "지윤"      # 값을 담는다
name == "지윤"     # 같은지 묻는다  →  True
```

## Scene s08 — 논리 연산자 3종 직관

- **sceneId**: `s08`
- **durationSeconds**: 12

**narration**:
마지막으로 논리 연산자 세 가지. 앤드는 둘 다 참이어야 참, 오어는 하나라도 참이면 참, 낫은 참과 거짓을 뒤집습니다. 예시로, 트루 앤드 폴스는 폴스, 트루 오어 폴스는 트루, 낫 트루는 폴스입니다.

**visual**:

- 0~4s: 화면 중앙에 3개의 큰 카드가 가로로 fade-in (좌→중→우 순서). 각 카드 상단에 연산자 이름, 하단에 한 줄 직관.
  - 카드 1: 헤더 `and` (violet-500). 한 줄 설명 "둘 다 참이어야 참". 예시 `True and False` → `False`.
  - 카드 2: 헤더 `or` (violet-500). 한 줄 설명 "하나라도 참이면 참". 예시 `True or False` → `True`.
  - 카드 3: 헤더 `not` (violet-500). 한 줄 설명 "뒤집기". 예시 `not True` → `False`.
- 4~12s: 각 카드의 예시 코드와 결과가 0.8초 간격으로 차례로 fade-in. 결과 (`True` / `False`) 는 모두 violet-600 굵은 글씨.

**onScreenCode**:

```python
True and False   # False  (둘 다 참이어야 참)
True or False    # True   (하나라도 참이면 참)
not True         # False  (뒤집기)
```

## Scene s09 — 시나리오: 시험 평균 계산 5줄 코드

- **sceneId**: `s09`
- **durationSeconds**: 30

**narration**:
이제 오늘 배운 것을 한번에 엮어 봅니다. 시나리오는 시험 평균 계산입니다. 국어, 영어, 수학 세 과목 점수를 입력받아, 합과 평균을 출력합니다. 한 줄씩 같이 가 보겠습니다. 첫 줄, 인풋 괄호 안에 국어 점수 안내문을 넣습니다. 그 결과를 인트 괄호로 감쌉니다. 그 값을 변수에 담습니다. 둘째, 셋째 줄도 같은 패턴으로 영어와 수학 점수를 받습니다. 넷째 줄에서 세 점수를 더해 합을 구하고, 다섯째 줄에서 합을 3으로 나눠 평균을 구합니다. 마지막으로 프린트 두 번으로 합과 평균을 화면에 보여 줍니다.

**visual**:

- 0~3s: 좌상단 작은 라벨 카드 "시나리오 — 시험 평균 계산" (violet-100 배경).
- 3~22s: 화면 좌측 60% 에 코드 에디터 패널. 아래 코드가 한 줄씩 type-on (각 줄 약 2~3초):
  ```python
  korean = int(input("국어: "))
  english = int(input("영어: "))
  math = int(input("수학: "))
  total = korean + english + math
  average = total / 3
  print("합:", total)
  print("평균:", average)
  ```

  - 한 줄이 type-on 될 때마다 그 줄이 0.3초 동안 violet-100 으로 깜빡 강조 후 흰 배경 복귀.
  - `int`, `input`, `print` 토큰은 violet-300, 연산자 (`+`, `/`) 는 violet-500.
- 22~27s: 우측 40% 콘솔에서 입력 시뮬레이션이 차례로 type-on. `국어: 80` → `영어: 90` → `수학: 70`. 각 입력은 1초 간격.
- 27~30s: 콘솔 하단에 결과 fade-in. `합: 240` 과 `평균: 80.0` 이 차례로 출력. 두 결과 숫자(240, 80.0) 는 violet-500 으로 펄스 한 번.

**onScreenCode**:

```python
korean = int(input("국어: "))
english = int(input("영어: "))
math = int(input("수학: "))
total = korean + english + math

# 입력: 80, 90, 70
```

## Scene s10 — Active Recall 2: `/` vs `//` 차이

- **sceneId**: `s10`
- **durationSeconds**: 12

**narration**:
다시 한번 잠깐. 7 나누기 2와, 7 슬래시 슬래시 2의 결과를 비교해 봅시다. 한 쪽은 3.5, 다른 한 쪽은 3입니다. 어느 쪽이 어느 결과일까요. (2초 정적) 슬래시 하나는 실수 나누기로 3.5, 슬래시 두 개는 몫만 가져와서 3입니다.

**visual**:

- 0~4s: 화면 좌우로 split. 좌측 위에 코드 `print(7 / 2)`, 우측 위에 코드 `print(7 // 2)`. 두 코드 아래에 큰 물음표 박스가 각각 fade-in.
- 4~6s: (2초 정적) 두 물음표 박스 옆에 작은 카운트다운 점 3개 violet-500 점등.
- 6~10s: 좌측 물음표가 `3.5` 로 swap, 우측 물음표가 `3` 으로 swap. 각각 scale up 0.3s.
- 10~12s: 두 결과 아래에 작은 설명 라벨 fade-in. 좌측 "/ — 실수 나누기", 우측 "// — 몫만".

**onScreenCode**:

```python
print(7 / 2)    # 3.5  (실수 나누기)
print(7 // 2)   # 3    (몫만)
```

**activeRecallBeat**:

- 질문: "`7 / 2` 와 `7 // 2` 의 결과는 각각 무엇일까요?"
- 정답: `3.5` / `3`
- 화면 표시 방식: 좌우 split, 각 물음표 박스가 2초 정적 후 결과로 swap.

## Production checklist

### voiceover_to_record (scene 별 narration 텍스트)

- **s01 (13s)**: 안녕하세요. 파이썬 기초 4강을 시작합니다. 3강까지는 우리가 직접 값을 변수에 적었습니다. 오늘은 사용자가 직접 값을 넣고, 그 값으로 계산까지 해 보겠습니다.
- **s02 (12s)**: 먼저 인풋 함수의 기본 형태입니다. 변수 이름을 적고, 등호를 쓰고, 인풋 괄호 안에 안내문을 넣습니다. 코드를 실행하면 안내문이 화면에 뜨고, 사용자가 답을 입력하면 그 값이 좌변 변수에 담깁니다.
- **s03 (35s)**: 여기서 가장 자주 막히는 지점입니다. 인풋 함수가 돌려주는 값은 항상 문자열입니다. 사용자가 숫자 3을 입력해도, 변수에 들어가는 건 따옴표가 붙은 문자 3입니다. 그래서 이런 일이 생깁니다. 첫째 값과 둘째 값을 받아서 더해 봅니다. 사용자가 3과 5를 입력했을 때, 우리가 기대한 건 8이지만 화면에는 35가 찍힙니다. 왜냐하면 문자 3과 문자 5를 더하는 건, 두 문자를 이어붙이는 동작이기 때문입니다. 이걸 진짜 숫자로 다루려면, 인트 괄호로 한 번 감싸 줍니다. 인트는 정수를 뜻하는 함수로, 문자 3을 숫자 3으로 바꿉니다. 이렇게 바꾼 뒤 더하면, 화면에 8이 찍힙니다.
- **s04 (15s)**: 이제 계산에 쓰는 산술 연산자 일곱 가지를 한번에 보겠습니다. 더하기, 빼기, 곱하기, 나누기 네 개는 학교에서 본 그대로입니다. 슬래시 두 개는 몫만, 퍼센트는 나머지, 별표 두 개는 거듭제곱입니다. 그리고 한 가지 규칙. 곱셈과 나눗셈이 덧셈과 뺄셈보다 먼저 계산됩니다. 헷갈리면 괄호로 묶어 주세요.
- **s05 (10s)**: 여기서 잠깐. 이 식의 결과는 무엇일까요. 2 더하기 3 곱하기 4. 잠깐 멈춰서 생각해 보세요. (2초 정적) 정답은 14입니다. 곱하기가 먼저 계산되어 3 곱하기 4는 12, 거기에 2를 더해 14가 됩니다.
- **s06 (15s)**: 다음은 비교 연산자입니다. 두 값을 견주는 데 쓰는 여섯 가지. 같다, 다르다, 크다, 작다, 크거나 같다, 작거나 같다. 여기서 중요한 점 하나. 비교의 결과는 그 자체로 값입니다. 트루, 또는 폴스 두 개 중 하나입니다. 예를 들어, 3이 5보다 크냐고 묻고 프린트로 출력해 보면, 화면에는 폴스가 찍힙니다.
- **s07 (10s)**: 헷갈리기 쉬운 한 가지. 등호 하나는 값을 담는다는 뜻입니다. 등호 두 개는 같은지 묻는다는 뜻입니다. 모양은 비슷해도 역할은 전혀 다릅니다.
- **s08 (12s)**: 마지막으로 논리 연산자 세 가지. 앤드는 둘 다 참이어야 참, 오어는 하나라도 참이면 참, 낫은 참과 거짓을 뒤집습니다. 예시로, 트루 앤드 폴스는 폴스, 트루 오어 폴스는 트루, 낫 트루는 폴스입니다.
- **s09 (30s)**: 이제 오늘 배운 것을 한번에 엮어 봅니다. 시나리오는 시험 평균 계산입니다. 국어, 영어, 수학 세 과목 점수를 입력받아, 합과 평균을 출력합니다. 한 줄씩 같이 가 보겠습니다. 첫 줄, 인풋 괄호 안에 국어 점수 안내문을 넣습니다. 그 결과를 인트 괄호로 감쌉니다. 그 값을 변수에 담습니다. 둘째, 셋째 줄도 같은 패턴으로 영어와 수학 점수를 받습니다. 넷째 줄에서 세 점수를 더해 합을 구하고, 다섯째 줄에서 합을 3으로 나눠 평균을 구합니다. 마지막으로 프린트 두 번으로 합과 평균을 화면에 보여 줍니다.
- **s10 (12s)**: 다시 한번 잠깐. 7 나누기 2와, 7 슬래시 슬래시 2의 결과를 비교해 봅시다. 한 쪽은 3.5, 다른 한 쪽은 3입니다. 어느 쪽이 어느 결과일까요. (2초 정적) 슬래시 하나는 실수 나누기로 3.5, 슬래시 두 개는 몫만 가져와서 3입니다.
- **s11 (14s)**: 오늘은 사용자에게 값을 받고, 그 값을 숫자로 바꿔 계산해서 화면에 보여줬습니다. 받기 — 인풋, 바꾸기 — 인트, 계산 — 산술 연산자, 보여주기 — 프린트. 이 네 단계가 한 호흡으로 무낍니다. 다음 강의는 조건문. 오늘 만든 비교 연산자의 결과로, 프로그램이 판단하게 만들어 봅니다.

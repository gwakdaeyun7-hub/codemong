> 다음 강의: 4강 "입력과 연산자".

## 1. 행동 가능 학습 목표

이 영상이 끝났을 때 학습자는 다음을 할 수 있어야 합니다.

> 이 강의는 학습자가 **처음으로 실제 파이썬 코드를 만나는 강의**입니다. 목표는 "한 줄을 외우기"가 아니라 "한 줄을 읽고 무슨 뜻인지 말할 수 있기".

1. **변수가 무엇이고, 왜 쓰는지 한 문장으로 설명할 수 있다.**
   ("값에 이름을 붙여 두고 나중에 그 이름으로 다시 꺼내 쓰기 위해서." 박스/이름표 비유로 충분.)
2. **`이름 = 값` 형태의 할당 한 줄을 읽고, 무엇이 변수 이름이고 무엇이 값인지 가리킬 수 있다.**
   (`name = "코드몽"` 을 보고 "왼쪽이 변수 이름, 오른쪽이 그 변수에 들어가는 값"이라고 구분 가능.)
3. **숫자(정수·실수) / 문자열 / 불린 세 가지 자료형을 보고 어느 종류인지 구분해 말할 수 있다.**
   (`5`, `3.14`, `"안녕"`, `True` 를 각각 숫자/숫자/문자열/불린으로 분류. 따옴표 유무·대문자 여부가 구분 단서임을 인지.)
4. **`print()` 한 줄을 직접 보면서, 변수에 담긴 값을 화면에 보여주는 가장 짧은 방법임을 설명할 수 있다.**
   (`name = "코드몽"` → `print(name)` 의 결과가 화면에 `코드몽` 으로 나오는 흐름을 따라갈 수 있음.)
5. **세 자료형을 각각 변수에 담고 `print()` 로 출력하는 4~6줄짜리 예제를 처음부터 끝까지 읽어내며 "어느 줄에서 무슨 일이 일어나는지" 말할 수 있다.**
   (직접 타이핑까지 요구하지는 않음. *읽고 따라갈 수 있음* 수준.)

> 영상에서 코드는 학습자가 직접 치지 않아도 됩니다. **읽고 이해할 수 있다**가 이 강의의 도착점.

## 3. 입문자가 자주 막히는 지점 / 오개념

3강 내용에서 새로 도출한 항목. 영상에서 짧게라도 짚고 넘어갈 가치가 있음.

1. **"변수 이름에도 따옴표를 붙여야 하는 줄 안다"** (`"name" = "코드몽"`)
   - 좌변은 *이름*, 우변은 *값*. 따옴표는 "이건 글자다"라는 표시이므로 변수 이름에 붙으면 더 이상 이름이 아니게 됨.
   - 영상에서: 첫 할당 한 줄을 보여줄 때 "왼쪽엔 따옴표가 없죠?"를 한 번 짚고 넘어갈 것. 길게 설명하지 말고 한 컷으로.

2. **"`True` / `False` 를 소문자로 쓴다"** (`true`, `false`)
   - Python은 대소문자를 구분하고, 불린은 첫 글자가 반드시 대문자(`True` / `False`). 다른 언어 경험이 있는 학습자가 자주 헷갈림.
   - 영상에서: 불린을 처음 보여주는 컷에서 "첫 글자 대문자, 따옴표 없음"을 한 문장으로 명시. 깊게 안 들어감.

3. **"숫자에 따옴표를 붙여도 그래도 숫자라고 생각함"** (`"5"` 와 `5` 의 차이)
   - 따옴표가 곧 "이건 글자(문자열)다"라는 표시. 따옴표가 붙는 순간 숫자처럼 보여도 자료형은 문자열로 바뀜.
   - 영상에서: 한 컷만. `5` 와 `"5"` 를 나란히 보여주고 "왼쪽은 숫자, 오른쪽은 글자입니다" 한 문장. 더하기 같은 연산 결과까지는 4강 몫이므로 깊게 안 들어감.

4. **"변수에 새 값을 할당하면 이전 값이 어딘가에 남아 있을 거라 기대"**
   - 같은 이름에 새 값을 할당하면 이전 값은 사라짐. "박스에 새 물건을 넣으면 이전 물건은 버려진다" 비유로 충분.
   - 영상에서: 같은 변수에 두 번 값을 할당하는 짧은 시범 한 컷. `print()` 로 두 번째 값만 나오는 것을 보여주며 짚을 것.

> 4개 모두 짧게 다루되, 비중은 1번·2번에 무게를 두고 3번·4번은 한두 문장으로.

## 4. Scope 제한 (이 영상에서 다루지 *않을* 것)

영상 대본 작성 시 아래 주제는 의도적으로 제외합니다. 다른 강의에서 다룹니다.

- **`input()` 함수, 산술/비교/논리 연산자** — 4강. 3강에선 *값을 담고 출력하기*까지만.
- **형 변환 함수(`int()`, `float()`, `str()`)** — 4강 또는 별도. "`5`와 `"5"`는 다르다"는 *구분*만 짚고, 변환 방법은 다루지 않음.
- **f-string, `.format()`, `%` 포맷팅** — 4강 이후. 3강의 `print()` 는 변수 하나 또는 값 하나를 그대로 출력하는 가장 단순한 형태만.
- **조건문 / 반복문** — 5·6강. 불린(`True`/`False`)을 다루더라도 "조건과 함께 쓰면 어떻게 된다"는 4·5강 몫이므로 짚지 않음.
- **리스트, 딕셔너리, 튜플, 집합** — 7·8강. 3강에선 단일 값을 담는 변수만.
- **`None` 타입, 타입 힌트(`name: str = ...`), 타입 어노테이션** — 입문 코스 범위 밖.
- **변수 명명 규약 상세(snake_case vs camelCase, PEP 8, 예약어 목록)** — 한 줄만(예: "영문 소문자로 시작하면 안전합니다") 짚고 넘어감. 깊게 들어가지 않음.
- **문자열 따옴표 종류 깊이 설명(`'`, `"`, `"""` 차이)** — "둘 중 어느 따옴표든 괜찮다" 한 문장만. 삼중 따옴표·이스케이프 같은 디테일은 별도 강의 몫.
- **`print()` 의 매개변수(`sep`, `end`, `file` 등)** — 함수 자체의 깊은 동작은 다루지 않음. "괄호 안에 넣은 값을 화면에 보여준다" 수준만.
- **`print` 와 `print()` 의 차이(괄호 유무가 함수 호출 vs 함수 자체 참조)** — 입문자에겐 인지 부하만 큼. 영상에선 다루지 않음. 4강 이후 함수 개념을 다질 때 자연스레 만남.

## 5. 학습 흐름 권장 (대본 작성자 참고용)

영상의 4단 구조(도입 → 개념 → 예시 → 정리)에 맞춘 분배 권장. 시간은 권장이며, video-script-writer 가 narration 길이에 맞춰 조정 가능.

| 구간 | 시간(대략) | 다룰 내용 |
|------|-----------|----------|
| 도입 | 0:00 ~ 0:20 | "2강에서 절차를 정리했다면, 3강은 그 절차에 등장하는 *값*을 코드로 담는 단계"라는 한 줄 전환. 오늘 다룰 세 가지(변수 / 자료형 / 출력) 예고. |
| 개념 | 0:20 ~ 1:10 | 변수의 정의와 `이름 = 값` 형태(목표 1, 2). 자료형 세 가지를 따옴표/대문자 단서와 함께 나열(목표 3). `print()` 한 줄의 의미(목표 4). |
| 예시 | 1:10 ~ 2:30 | 한 시나리오를 잡아 4~6줄짜리 예제 작성·읽기(목표 5). 코드를 한 줄씩 따라가며 오개념 1·2 자연스럽게 삽입. 같은 변수에 새 값을 할당해 보여주는 짧은 컷으로 오개념 4 짚기. 숫자 vs `"숫자"` 비교 컷으로 오개념 3을 한 번만. |
| 정리 | 2:30 ~ 3:00 | "값에 이름을 붙이고 → 자료형을 구분하고 → 화면에 출력했다" 세 줄 요약. 다음 강의 hook(4강 "입력과 연산자"로 자연스레 넘어감). |

### 일상 시나리오 후보 (대본 작성자 선택용)

세 자료형(숫자·문자열·불린)을 한 예제 안에 자연스럽게 등장시키는 한국 입문자 친화 시나리오:

- **자기소개 카드**: 이름(문자열) + 나이(숫자) + 학생 여부(불린)를 변수에 담고 `print()` 로 한 줄씩 출력. → **세 자료형이 가장 균형 있게 등장**. 일상 친숙도 높음. **추천.**
- **장바구니 한 줄 결제**: 상품명(문자열) + 가격(숫자) + 할인 적용 여부(불린)를 변수에 담고 출력. → 숫자 비중이 약간 더 큼. 후속 4강(연산자)과 자연스럽게 연결되는 장점.
- **오늘 날씨 메모**: 도시(문자열) + 기온(숫자, 실수까지) + 우산 챙길지(불린)를 변수에 담고 출력. → 실수형(`float`)을 자연스레 등장시키기 좋음. 다만 "우산"이 살짝 작위적일 수 있음.

세 후보 중 **자기소개 카드**를 우선 추천. 세 자료형이 가장 균형 있게 들어가고, 학습자가 직접 자기 이름·나이로 응용하기도 자연스러움. video-script-writer 가 시나리오 결을 보고 최종 선택.

> 한 영상 안에서 **한 시나리오만** 끝까지 다룰 것. 두 개 이상 쓰면 시간 부족 + 인지 부하 증가.
